Leaf bundles 与 Branch bundles

我认为，理解 Leaf bundle 与 Branch bundle 是使用Hugo比较重要的一步。

什么是bundle?

Bundle就是Hugo组织页面的方式

从index.md 与 _index.md说起

.
└── content
    └── about
    |   └── index.md  // <- https://example.com/about/
    ├── posts
    |   ├── firstpost.md   // <- https://example.com/posts/firstpost/
    |   ├── happy
    |   |   └── ness.md  // <- https://example.com/posts/happy/ness/
    |   └── secondpost.md  // <- https://example.com/posts/secondpost/
    └── quote
        ├── first.md       // <- https://example.com/quote/first/
        └── second.md      // <- https://example.com/quote/second/

当使用index.md，当前文件夹就叫 leaf bundle

当使用_index.md, 当前文件夹就叫 branch bundle

leaf bundle ，我的理解就是用一个文件夹来装当前页面的所有内容，本博文便是在如下的一个 leaf bundle：

关于hugo使用的一些心得
├──index.md
├──我就在这儿上学啊.jpg

Leaf bundles 的使用

也就是说，你本篇博文的主要内容在index.md里写，文件夹名称是你本篇博文的标题，文件夹里有image, audio, pdf等页面资源

引用图片

在markdown中:

![我就在这儿上学啊](我就在这儿上学啊.jpg)
hugo new
hugo new posts/directory-name/index.md

Branch bundles 的使用

Branch bundles 往往与 list Page 的概念绑定，即_index.md用来修饰一个 list Page

比如本博客主页“封面致敬我最爱的乐队——Architects” 就是在content/_index.md里面写的

总结

举例说明，本博客中的About页面，我们想把他布局到yoursite.com/about-us/

以下三种方式都可以：

  • content/About.md
  • content/About/index.md
  • content/About/_index.md

应该选用哪种呢？

  1. 你这个页面有子页面吗？比如，你如果有以下页面
  • yoursite.com/About/
  • yoursite.com/About/mygirlfriend/
  • yoursite.com/About/myself/

那么你就要用 content/About/_index.md, 因为这告诉 Hugo About是一个list page，官方文档参考

对于list page, Hugo会使用list.html的模板

  1. 我确实有子页面，但我又不想我的子页面是单独的页面

当你只是想把页面分成几个部分，又不是想要几个单独的页面，你可以采用如下：

content
├── About
│   ├── index.md
│   ├── mygirlfriend
│   │   ├── img.jpg
│   │   └── introduction.md
│   ├── myself
│   │   ├── img.jpg
│   │   └── introduction.md
│   └── ...
  1. 我就是要一个About，没有花里胡哨的

好的。出于整齐性，你可以还是content/About/index.md；当然，content/About.md也是可以的。

这两种，Hugo 都会使用single.html的模板
